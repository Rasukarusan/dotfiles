カレントディレクトリに存在する全ての画像またはPDFから経費情報を読み取り、整理されたMarkdown表を出力するスキルです。

## 処理フロー

1. カレントディレクトリの画像/PDFファイルを全て検出
2. 各ファイルを読み取り、経費情報を抽出
3. `hyou.md`に表を出力
4. ファイル名にprefixを追加してリネーム
5. 画像ファイルを圧縮

## 出力フォーマット

`hyou.md`に以下の表を出力する：

```
|品名|勘定科目|空白|取引先|金額|日付|税区分|収支区分|
```

## 各カラムの判定ルール

### 品名

- カフェ（コメダ、高倉町、むさしの森、星乃、タリーズ、ドトール、ベローチェ等）→ `カフェ`
- ただしコーヒーチケットの場合 → `コーヒーチケット`
- 書籍1冊 → `本_書籍名`
- 書籍複数 → `本_書籍名1_書籍名2`
- その他 → レシートの内容から適切に判断

### 勘定科目

以下から選択：

| 勘定科目 | 判定基準 |
|---|---|
| 交際費 | 飲食店での会食、贈答品 |
| 消耗品費 | 事務用品、PC周辺機器等 |
| 新聞図書費 | 書籍、電子書籍、新聞 |
| 通信費 | インターネット、電話、サブスク通信系 |
| 研修費 | Udemy、セミナー、研修 |
| 雑費 | カフェ（作業場所としての利用） |
| 旅費交通費 | 電車、新幹線、タクシー |
| 会議費 | 打ち合わせ時の飲食 |
| 車両費 | ガソリン、車検、駐車場 |

### 取引先

以下のリストから最も近いものを選択する。存在しない場合はレシート記載のまま出力：

Microsoft, 文教堂, Udemy, 蔦屋書店, コメダ珈琲店, むさしの森珈琲, アニール, OpenAI, 車検のコバック, サクラ書店, 有隣堂, コナズ珈琲, 高倉町珈琲, 東海旅客鉄道株式会社, IVY PLACE, ドトール, DOUTOR, 8LOUNGE, 珈琲館OB, Divo, TSUTAYA, モキチ, さくら書店, 伊勢治書店, 三省堂書店, ジュンク堂書店, 有限会社珈琲茶館, タリーズコーヒー, カフェベローチェ, ディーンカフェ, 東京堂書店, 星乃珈琲店, Enejet, BIG ECHO

### 税区分

以下から選択。不明な場合は空欄：
- `課対仕入10%`
- `課対仕入8%`（軽減税率対象：飲食料品のテイクアウト等）
- `対象外`

### 収支区分

必ず `支出` とする。

## ファイルリネーム

勘定科目に応じて画像ファイル名にprefixを追加する：

| 勘定科目 | prefix |
|---|---|
| 交際費 | `gohan_` |
| 消耗品費 | `syoumouhin_` |
| 新聞図書費 | `book_` |
| 通信費 | `net_` |
| 研修費 | `other_` |
| 雑費 | 店名（例：`komeda_`, `tullys_`, `hosino_`） |
| 車両費 | `gasorin_` |
| 旅費交通費 | `train_` |
| 会議費 | `gohan_` |

雑費の店名prefixは、店名をローマ字小文字にして `_` を付ける。

リネームはスクリプトで実行：

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/rename-files.sh <prefix> <ファイルパス>
```

## 画像圧縮

全ての画像ファイルをimagemagickで品質30%に圧縮する。スクリプトで一括実行：

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/compress-images.sh <対象ディレクトリ>
```

- jpeg, png, heic, webp が対象
- PDFは圧縮対象外
- 圧縮前後のサイズと削減率が表示される
